### 1️⃣ 버그 수정 (Commit: 303994e)
**f64 산술 연산 & 배열 요소 수정 스택 순서 버그 고정**

#### 문제
- VM의 f64 산술 연산 (ADD_F64, SUB_F64, MUL_F64, DIV_F64, MOD_F64, NEG_F64)이 구현되지 않음
- Compiler의 배열 요소 할당 (arr[1] = 99)이 스택 순서 오류로 작동 안 함

#### 해결
```
✅ vm.ts (line 225-263)
   - 6가지 f64 연산 전부 구현
   - i32 연산과 동일 로직 (JS는 number 통일)

✅ compiler.ts (line 1286-1303)
   - 스택 순서 수정: 값→배열→인덱스 → 배열→인덱스→값
   - VM ARRAY_SET의 pop 순서에 맞춤
```

**영향**:
- struct Point { x: f64, y: f64 } 테스트 통과 ✅
- 배열 요소 수정 (arr[1] = 99) 작동 ✅
- VM 테스트 시간: 30초+ → **17.2초** (45% 단축)
