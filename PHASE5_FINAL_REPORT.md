## 📊 주요 설계 결정사항

### 1. 토큰 구분 (2자 연산자)

**문제**: "=" vs "==" 구분

**해결책**: Lookahead (다음 문자 미리 확인)
```freelang
if c == "=" {
  if li + 1 < length(source) && char_at(source, li + 1) == "=" {
    // TOK_EQ_EQ
  } else {
    // TOK_EQ
  }
}
```

### 2. 평행 배열 구조

**문제**: 타입 체커가 혼합 타입 배열 거부

**해결책**: 별도 배열 유지
```freelang
var lexeme_types = []   // [TOK_VAR, TOK_IDENT, ...]
var lexeme_values = []  // ["var", "x", ...]
```

### 3. 백패칭 (Backpatching)

**문제**: 점프 명령어 생성 시 목표 주소 미정

**해결책**: 두 번 순회
```
1단계: JMP 생성 (오프셋 = 0)
2단계: 목표 코드 생성
3단계: 실제 오프셋으로 수정
```

### 4. 루프 처리

**해결책**: JMP로 루프 시작으로 돌아가기
```
loop_start: LOAD x
            CMP x < 10
            JMP_IF_FALSE exit
            [루프 본체]
            JMP loop_start
exit:       [다음 코드]
```
