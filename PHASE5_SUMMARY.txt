================================================================================
                            ARCHITECTURE
================================================================================

Design Pattern:
  Each command follows strict pattern:
  1. Input validation
  2. Flag parsing
  3. Configuration loading (auth check)
  4. HTTP client creation
  5. Async API call
  6. Result handling with Result<T>

Error Handling:
  - Result<T> type for all operations
  - Match pattern matching
  - Comprehensive error messages
  - Safe deletion with --force flag

Output Formats:
  - text (default): formatted tables/boxes
  - json: JSON output
  - yaml: YAML output
  - --json, --yaml flags

Caching:
  - HTTP response caching with TTL
  - Cache hit on list queries
  - Cache invalidation on write
  - Configurable TTL (300s default)

Idempotency:
  - gogs repo ensure: create or update
  - gogs batch ensure: full state sync

Safety Features:
  - Confirmation required for destructive ops
  - --force flag for implicit confirmation
  - Input validation on all fields
